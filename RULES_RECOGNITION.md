> **参照元:** `ルールブック/ルールブック_表.png`・`ルールブック/ルールブック_裏.png`、および `カード/` 内の各カード画像（2025年版）

---

## 1. ゲーム概要

| 項目       | 内容                                                                                                     |
| ---------- | -------------------------------------------------------------------------------------------------------- |
| タイトル   | 定時退社（Leaving work on time）                                                                         |
| 人数       | 3〜6人                                                                                                   |
| プレイ時間 | 3〜10分                                                                                                  |
| 対象年齢   | 7歳〜                                                                                                    |
| ジャンル   | ババ抜き系・手札管理・心理戦                                                                             |
| テーマ     | ブラック企業の社員が仕事（カード）を他人に押し付け、誰より早く手札をなくして「定時退社」することを目指す |

**勝利の考え方（2系統）**

1. **通常勝利:** 手札をすべて捨てて「退社」する（脱落＝勝ち）
2. **労基勝利:** 他人の手札から「残業」を暴き、即座にゲームを終了させる（労基を出した人が大勝利）

**敗北の考え方**

1. 最後まで「残業」カードを持ち続ける（通常終了時）
2. 「労基」効果で「残業」が公開された時点で、その持ち主が即敗北（強制終了）

## 3. 用語集（ルールブック裏より）

| 用語             | 意味                                                                                                     |
| ---------------- | -------------------------------------------------------------------------------------------------------- |
| **手札**         | 各プレイヤーが持つ仕事カード                                                                             |
| **公開**         | カードを表向きにして全員が見られる状態にし、その後持ち主の手札に戻す                                     |
| **見る**         | 行動するプレイヤーのみが内容を確認し、他プレイヤーには見えない。確認後、持ち主の手札に戻す               |
| **選ぶ**         | カードを出したプレイヤーが対象（**社員**または**カード**）を指定する。**自分を対象に選ぶことはできない** |
| **〜してもよい** | 任意（やらなくてもよい）                                                                                 |

## 4. 準備

1. 「残業」1枚と、その他48枚の仕事カードを合わせてシャッフルする
2. 全カードを全プレイヤーに配る
   - 基本は均等配布
   - **端数**がある場合、**初手プレイヤーから時計回り**に1枚ずつ追加で配る
3. **開始時ペアの扱い:** 配布直後に手札にペアがあっても、**その場では捨てられない**。自分のターンが来るまで保持する
4. **初手プレイヤー:** **ランダム**で決定し、そこから時計回りにプレイを開始する

> 物理版ルールブックの「直近で残業した人から」は、オンライン初回ではランダムに置き換える。

## 5. ターンの流れ

各ターン、プレイヤーは**必ず以下の順番**で行動する:

### 5.1 カードを引く（必須）

- **右隣の社員**の手札から1枚引く
- 右隣がすでに退社している場合は、さらに右（時計回り）の在籍社員から引く
- **引くカードは、引く側（ターンのプレイヤー）が選ぶ。** 相手の手札が裏向きで見えない状態から1枚を取る（ババ抜きの基本）。オンラインでは**ランダム**で1枚を自動選択する

**オンラインでの隣の定義**

- 全員が**円形に座る**イメージで座席順を固定する（§10.6）
- 「右隣」= 画面上の時計回りの次のプレイヤー（物理版と同じ）
- 引く方向・渡す方向（情報共有の左隣など）は、この座席順に一貫して適用する
- 手動操作時はプレイヤーが選択。約20秒未操作・切断・放置時は**ランダム**代行

### 5.2 仕事カードを出す（任意）

- 手札に**同じ種類のカードが2枚以上**あれば、**1ペア**を場に出してよい
- 出したペアの**効果を発動**する（効果なしのカードもある）
- **ペアを出さなくてもターンは終了できる**（戦略的に温存可能）
- 1ターンに出せるペアは通常1組。ただし「エナドリ」効果により追加が可能（後述）

### 5.3 退社（脱落＝勝ち）

- 手札が**0枚**になったプレイヤーは**退社**（勝利）し、ゲームから抜ける
- 退社したプレイヤーは**すべてのカード効果の対象外**になる

### 5.4 効果解決と退社判定（重要）

| 原則                     | 内容                                                                                       |
| ------------------------ | ------------------------------------------------------------------------------------------ |
| **退社判定のタイミング** | カード効果の**処理がすべて終わった後**に、各プレイヤーの手札枚数を判定する                 |
| **チェーン中の退社**     | **なし**。効果処理の途中で手札が0枚になっても、その時点では退社しない                      |
| **例外: 労基×残業**      | 「残業」が労基で公開された場合は、効果解決の一部として**即敗北・ゲーム終了**（退社とは別） |

**実装フロー（1効果あたり）**

```
1. カード効果を最後まで解決する（手札の移動・公開・交換など）
2. 全在籍プレイヤーの手札枚数をチェック
3. 0枚のプレイヤーを退社処理する（複数同時もあり得る）
```

### 5.5 対象選択の共通ルール

| 状況                | 処理                                                                     |
| ------------------- | ------------------------------------------------------------------------ |
| 通常                | カードを出したプレイヤーが、対象**プレイヤー**および対象**カード**を選ぶ |
| 有効な対象が1人のみ | そのプレイヤーを**自動対象**とし、他のプレイヤーは選択不可               |
| 自分                | 常に対象に選べない                                                       |

## 6. 特殊カード: 残業（ババ）

| 項目         | 内容                                                                                                 |
| ------------ | ---------------------------------------------------------------------------------------------------- |
| 枚数         | 1枚                                                                                                  |
| ペア         | **ペアを作れない**（通常のペア捨て不可）                                                             |
| 場への放出   | **通常は場に出せない**                                                                               |
| 敗北条件①    | ゲーム通常終了時、最後まで手札に残っている                                                           |
| 敗北条件②    | 「労基」により**公開された時点で即敗北**                                                             |
| 会議との関係 | 「会議」が場に出されたとき、手札に残業を持つ社員は**「持っています」と宣言**する（**嘘不可・強制**） |

## 7. 各カードの効果（ペアを場に出したとき発動）

### 7.1 ノルマ

- **効果:** なし
- **補足:** ゲーム内で最も役に立たないカード

### 7.2 労基（労働基準監督署）

- **効果:** 社員を1人選び、裏向きの手札から1枚選んで**公開**する
- **選択者:** 対象プレイヤー・公開する1枚とも、**労基を出したプレイヤー**が選ぶ
- **公開時の追加判定:**
  - 公開されたカードが「残業」→ その持ち主は**即敗北**、ゲーム強制終了
  - 公開されたカードが「パワハラ」→ パワハラの公開時効果を発動（後述）。通常の「手札に戻す」は適用されない
- **補足:** 上記以外のカードは、全員に見せたあと持ち主の手札に戻す（公開の定義）

### 7.3 飲み会

- **効果:** **次の手番の社員**は、仕事カードを場に出すことができない
- **範囲:** **すべてのペア放出が不可**（エナドリで増えた追加ペアも含む）
- **補足:** 次の社員は「カードを引く」だけ行い、ペアを出さずにターン終了する
- **フレーバー:** 「今日空いてるよね？（拒否権なし）」

### 7.4 社内恋愛

- **効果:** 社員を1人選び、**互いの手札をすべて見せ合う**
- **補足:** 一時的に手札を見せ合い、確認後それぞれの持ち主に戻す（公開ではなく、2人だけが全手札を知る）

### 7.5 新人教育

- **効果:** 社員を1人選び、手札を2枚**見る**（見られる枚数が2未満ならあるだけ見る）。そのうち1枚を自分の手札に**加えてもよい**
- **加えない選択:** **見るだけで終了可**（カードを加えなくてよい）
- **選択者:** 対象プレイヤー・見るカード・加えるカードとも、**新人教育を出したプレイヤー**が選ぶ
- **退社判定:** 効果処理完了後に手札枚数を判定（§5.4）

### 7.6 情報共有

- **効果:** **全員**が手札から1枚選び、**左隣の社員**に渡す
- **解決タイミング:** 全員が渡すカードを選び終わった時点で、**一斉に交換**する
- **補足:** 自分が渡すカードを選ぶとき、まだ受け取るカードの中身は確認できない

**オンライン実装**

| 状況           | 処理                                                    |
| -------------- | ------------------------------------------------------- |
| 全員が選択完了 | 同時に左隣へ1枚ずつ渡す                                 |
| タイムアウト   | 未選択のプレイヤーは**ランダム**で1枚を自動選択して交換 |

### 7.7 取引

- **効果:** 社員を1人選び、互いに手札から1枚ずつ選んで**交換**する
- **解決タイミング:** 双方が選び終わったら**同時に交換**する
- **補足:** 裏向きで交換。受け取るまで中身は見えない

### 7.8 エナドリ

- **効果:** **このターン**、追加でもう1組ペアの仕事カードを場に出してもよい
- **スタック:** 追加で出したペアが再び「エナドリ」なら、効果は**さらにスタック**する。**上限なし**
- **例:** エナドリを1回発動 → 通常1組＋追加1組で**計2組（4枚）**出せる。エナドリを2回スタック → 計3組（6枚）出せる
- **補足:** 他に出せるペアがなくても、このカード自体は場に出せる

### 7.9 会議

- **効果:** 手札に「残業」を持つ社員は**「持っています」と宣言**する
- **嘘は不可:** 残業を持っている場合、**必ず正直に宣言**する（虚偽宣言はルール違反）
- **補足:** 会議を出した本人も、残業を持っていれば宣言必須。中身は見せない（宣言のみ）

**オンライン実装**

- 残業を持つプレイヤーには**強制的に宣言UIを表示**する（スキップ・嘘不可）
- 残業を持たないプレイヤーは宣言不要

### 7.10 パワハラ

- **通常効果:** 社員を1人選び、自分の手札から1枚渡す
- **選択者:** 対象プレイヤー・渡すカードとも、**パワハラを出したプレイヤー**が選ぶ
- **労基で公開されたとき:** 公開されたこの「パワハラ」カードは、**労基を場に出したプレイヤー**の手札に移動する（元の持ち主の手札には戻らない）
- **退社判定:** 効果処理完了後に手札枚数を判定（§5.4）

### 7.11 タバコ休憩

- **効果:** 手札に「タバコ休憩」を持つ**全社員**は、それを**すべて**場に出す
- **発動順:** **全員同時**に場へ出す（オンラインでは自動処理でよい）
- **退社判定:** 効果処理完了後に手札枚数を判定（§5.4）

## 8. ゲーム終了と勝敗

ゲームは以下のいずれかが発生した**瞬間**に終了する。

### 8.1 パターンA: 通常終了（退社）

| 条件 | 最後に1人だけが手札を持っている状態になる |
| ---- | ----------------------------------------- |
| 勝者 | すでに退社した全員（退社順が順位）        |
| 敗者 | 最後まで「残業」を持っている1人           |

### 8.2 パターンB: 強制終了（労基摘発）

| 条件             | 「労基」効果で「残業」が公開されたとき |
| ---------------- | -------------------------------------- |
| 大勝利           | 「労基」を場に出したプレイヤー         |
| 大敗北           | 「残業」の持ち主                       |
| すでに退社した人 | **共同責任**（引き分け扱い）           |

### 8.3 終了後（演出）

敗者は、片付け・次のゲーム準備・SNSでの感想投稿など「残業」的な雑務を担当する（ルール上の罰ゲーム／フレーバー）

## 10.6 オンライン運用（タイムアウト・切断・座席）

### 行動待ちの基本方針

| 項目               | 方針                                                                    |
| ------------------ | ----------------------------------------------------------------------- |
| ターン制限         | **カウントダウン式の厳密な制限時間は設けない**                          |
| 未操作の検知       | 約 **20秒** 行動がなければ、サーバーが自動処理する                      |
| 自動処理の原則     | **ランダムでできる操作はすべてランダム**で実行                          |
| ランダム不可の場合 | スキップ or ルール上の自動処理（例: 会議の強制宣言、対象1人の自動選択） |

### 状況別の自動処理

| 状況                         | 処理                                                                   |
| ---------------------------- | ---------------------------------------------------------------------- |
| ターン中・カードを引く       | 右隣（在籍者）の手札から**ランダム**に1枚引く                          |
| ターン中・ペアを出す（任意） | 出せるペアがある場合は**ランダム**で1組出す。なければスキップ          |
| 対象プレイヤー選択           | 有効な対象から**ランダム**（1人のみなら自動）                          |
| 対象カード選択               | 候補から**ランダム**                                                   |
| 情報共有                     | 手札から**ランダム**に1枚を左隣へ                                      |
| 取引                         | 双方とも未操作なら、各自の手札から**ランダム**に1枚を同時交換          |
| 新人教育                     | 見るカードは**ランダム**。加えるかは**ランダム**（加えないも可のため） |
| 会議                         | 残業保持者は**強制宣言**（ランダムではない）                           |
| **切断中**                   | 上記と同様に、そのプレイヤーの操作を**ランダム**で代行                 |
| **退室・放置**               | 切断と同様、**ランダム**で代行してゲームを継続                         |

> 再接続したプレイヤーは、代行中に変化したゲーム状態（手札・場・ターン）をそのまま引き継ぐ。

### 座席順の決め方（ルーム設定）

ルーム作成時に以下の **2モード** から選択する。

| モード         | 内容                                                         |
| -------------- | ------------------------------------------------------------ |
| **ランダム**   | ゲーム開始時にプレイヤー順をシャッフルして座席を決定         |
| **ホスト指定** | ホストが開始前に座席（並び順）を**ドラッグ等で並べ替え**可能 |

- どちらのモードでも、ゲーム中の座席順は固定（途中で入れ替えない）
- 右隣・左隣・引く方向・情報共有の渡し方向は、この座席順に基づく

## 11. Webアプリ移植の可否判断

### 結論: **移植可能（推奨度: 高）**

物理版のメカニクスは、デジタル化に適した構造になっている。

### 移植しやすい理由

| 観点       | 評価                                            |
| ---------- | ----------------------------------------------- |
| ゲーム構造 | ターン制・有限状態・明確な終了条件              |
| 情報管理   | 手札の公開/非公開はサーバー側で自然に管理できる |
| カード効果 | 12種類+ババと効果数は実装規模として妥当         |
| プレイ時間 | 3〜10分。1セッション完結型でUX設計しやすい      |
| 人数       | 3〜6人。オンラインルーム向き                    |
| アセット   | カード画像・ルールブックが既にデジタル化済み    |

### 実装で工夫が必要な箇所

| 機能                 | 難易度 | 内容                                                    |
| -------------------- | ------ | ------------------------------------------------------- |
| 非同期マルチプレイ   | 中     | ターン制＋WebSocket。未操作20秒・切断時はランダム代行   |
| 隠し情報UI           | 中     | 手札、公開/見る/交換の段階的表示                        |
| 同時選択（情報共有） | 中     | 全員選択待ち → 一斉交換。タイムアウトはランダム自動選択 |
| 対象選択UI           | 低〜中 | 労基・新人教育・取引などのターゲット選択                |
| 効果チェーン         | 中     | 効果完了後の一括退社判定。タバコ休憩は全員同時に場へ    |
| 宣言（会議）         | 低     | 残業保持者への強制宣言UI（嘘不可をサーバーで担保）      |
| ハウスルール         | 低     | 会話ルールはルーム設定として切り替え可能にするとよい    |

### 移植の推奨アーキテクチャ（概要）

```
[クライアント] ←WebSocket→ [ゲームサーバー]
                              ├── Room（3〜6人）
                              ├── GameState（手札・山・場・ターン）
                              └── EffectResolver（カード効果の順次/割り込み処理）
```

**最小実装スコープ（MVP）**

1. 同室リアルタイム対戦（3〜6人）
2. 全カード効果の自動処理
3. 退社・労基即敗北の自動判定
4. 座席順（ランダム or ホスト指定）と隣の定義

**あるとよい拡張**

- 観戦モード
- リプレイ
- ボット（人数不足時）
- ハウスルール設定（ブラフ可/不可）
- 拡張セット対応（`定時退社拡張` フォルダが別途存在）

### 移植不可・困難な要素

**なし（致命的な阻害要因はない）**

テーブルトーク・ブラフは「公式ルール外」のため、デジタルで再現しなくてもゲームとして成立する。チャットやスタンプで代替可能。
